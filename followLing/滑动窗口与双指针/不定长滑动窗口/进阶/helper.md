## 2779. 数组的最大美丽值

这道题元素顺序对答案没有影响, 可以先排序

拿示例1做例子: 数组 nums [4,6,1,2], 对于其中的每一个数 nums[i] 都可以是 [nums[i] - k, nums[i] + k]
根据范围可以画出: 
<img width="1691" height="711" alt="Image" src="https://github.com/user-attachments/assets/961d82c4-55b5-4081-82ec-c0fa08f5468b" />

只要两个区间有重叠部分(大数的左边界小于小数的右边界)就可以判定: 这两个数可以通过 加/减 k 来达到相同的值
对于下标为 left 和 right 的数来说就是: nums[right] - k <= nums[left] + k
可以转换为 nums[right] - nums[left] <= 2 * k
这就得到了窗口的边界

## 2831. 找出最长等值子数组

题目关键 "从 nums 中删除最多 k 个元素后，返回可能的最长等值子数组的长度", 换句话说就是, 对于元素 x 来说, 一段以 x 开头并且以 x 结尾的连续子数组, 其中有多少元素是不等于 x 的, 这个数量与 k 作比较, 小于 k 则满足要求.

具体点, 拿示例一做例子就是: [1,3,2,3,1,3]
单独看数字3, 一段以 3 开头并且以 3 结尾的连续子数组就是: [3,2,3,1,3], 其中不等于 3 的元素有 2 和 1,共 2 个. 题目中最多可以删除k 个, 示例一的 k = 3, 所以[3,2,3,1,3]满足要求

先使用 map 做分类, 得到: { 1 => [ 0, 4 ], 3 => [ 1, 3, 5 ], 2 => [ 2 ] }
键为 key, 值为 value
对其中的每一项做不定长滑动窗口, 窗口条件是: 
对于窗口的left, right. value[right] - value[left] + 1 是连续子数组的长度 
right - left + 1 是这一段中包含的元素的个数, 这两个值相减就是需要删掉的数量, 需要小于等于 k, 可以得到公式: 
value[right] - value[left] + 1 - (right - left + 1) <= k
